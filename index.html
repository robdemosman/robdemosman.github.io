Git?

Ik ben zelf nieuw met het fenomeen Git en probeer hier de kernpunten samen te vatten tijdens mijn ontdekkingstocht.
Als bron heb ik dit boek gebruikt: Pro Git. Deze website is tevens voor mij een test om wat te oefenen met Git.

Basis

In grote lijnen komt het erop neer dat je met git aanpassingen - die je aan bv een website of een programma dat je aan het coderen bent - kunt bijhouden en je op elk gewenst moment deze weer ongedaan kunt maken en/of kunt opzoeken wat je ooit hebt veranderd. Elke keer dat je commit (je wijzigingen opslaat) wordt er een snapshot gemaakt van de huidige staat van je project. Met git werk je in principe lokaal op je computer en als je klaar bent met een bepaalde stap binnen je project dan kun je via de opdracht commit deze aanpasingen opslaan in je project.

Dit opslaan gaat in enkele fasen:

Ctrl+s
Als je met bv Atom werkt dan sla je met Ctrl+s je wijzigingen lokaal op. Atom signaleert dat je nu iets veranderd hebt en het gewijzigde bestand staat nu bij de unstaged changes.
Stage
Als je nu bv in meerdere bestanden aanpassingen hebt gedaan en je wilt gaan testen of het allemaal functioneert dan moet je deze eerst met het commando Stage deze klaar zetten om je origineel project bij te werken.
Commit
Nu kun je met Commit deze wijzigingen nog met een uitleg wat je gedaan hebt als 1 geheel klaarzetten om op te slaan. Dit gebeurt allemaal nog lokaal.
Push
Als laatste druk je op Push en dan gaan de wijzigingen naar de server waar het originele project staat.
